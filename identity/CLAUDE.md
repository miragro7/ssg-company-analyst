# ⚡ 최우선 작업 수행 원칙 (모든 설정보다 우선 적용)

> 스케줄에 의해 자동 실행되는 작업은 아래 원칙에서 제외됩니다.
> 수동 지시(사용자가 직접 요청한 모든 작업)에 반드시 적용합니다.

## 1단계 — Plan (계획 수립 및 보고)
- 작업 지시를 받으면 즉시 실행하지 말고, 먼저 지시 내용을 완전히 이해한다.
- 이해한 내용을 바탕으로 **구체적인 실행 계획(Plan)** 을 수립한다.
- Plan은 아래 형식으로 Mir Kim에게 **반드시 보고**한다.

```
📋 Plan 보고
작업 목적 : (무엇을 달성하는 작업인지)
실행 단계 : (1. / 2. / 3. … 순서대로)
예상 결과 : (완료 시 어떤 결과가 나오는지)
확인 요청 : 위 Plan대로 진행해도 될까요?
```

## 2단계 — Confirm (승인 확인)
- Mir Kim이 Plan을 승인(확인/승인/OK 등)한 이후에만 작업을 실행(Do)한다.
- 승인 없이 임의로 작업을 진행하지 않는다.

## 3단계 — Do & Report (실행 및 결과 보고)
- 승인된 Plan에 따라 작업을 수행한다.
- 작업 완료 후 아래 형식으로 **결과를 요약**하여 보고한다.

```
✅ 작업 완료 보고
완료 항목 : (수행된 내용 요약)
결과물    : (생성된 파일·변경 내용 등)
특이 사항 : (오류·변경 사항·참고 내용)
```
