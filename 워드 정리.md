# 워드 정리 기능

> **범위:** 뉴블로그 텔레그램 봇  
> **실행 주체:** 미르(bot.py)  
> **처리 스크립트:** `/home/ubuntu/뉴블로그/word_organizer.py`

---

## 텔레그램 실행 명령어

```
워드 정리
```

---

## 동작 흐름

```
텔레그램 입력: "워드 정리"
  → bot.py _WORD_ORG_RE 패턴 감지
  → 봇 응답: "워드 정리가 필요한 파일을 첨부해주세요~"
  → 사용자가 파일 전송 (.docx / .html / .txt)
  → word_organizer.py 실행
      1. 파일 내용 추출 (포맷별 파서)
      2. 제목 + 섹션 + 불릿 구조로 파싱
      3. content.json 생성
      4. sds-word-writer generate.py 호출 → 새 Word 파일 생성
  → 생성된 Word 파일을 텔레그램으로 전송
```

---

## 지원 파일 포맷

| 확장자 | 처리 방식 |
|--------|-----------|
| `.docx` | python-docx로 단락·스타일 추출, 볼드 단락은 헤딩으로 인식 |
| `.html` / `.htm` | BeautifulSoup으로 h1~h6(헤딩), p/li/td(본문) 추출 |
| `.txt` | 줄별 파싱, 번호/기호/대문자 패턴으로 헤딩 자동 감지 |

---

## 출력 Word 파일 스타일

스드스 기업 분석 리포트와 동일한 sds-word-writer 스타일 적용:

| 항목 | 값 |
|------|----|
| 폰트 | 바탕체 |
| 장체 | 95% |
| 본문 크기 | 14pt |
| 줄간격 | single (line=240) |
| 섹션 기호 | Ⅰ Ⅱ Ⅲ … (로마 숫자) |
| 불릿 종류 | dash (- ) 기본, dot (·) 서브 항목 |
| 페이지 번호 | 하단 중앙 |
| 마감 | - 이  상 - (우측 정렬) |

## 파일 구조

```
/home/ubuntu/뉴블로그/
├── word_organizer.py       — 파일 추출 + content.json 변환 + generate.py 호출
├── bot.py                  — _WORD_ORG_RE 트리거 + _word_org_state 상태 + handle_document 핸들러
└── skills/sds-word-writer/scripts/generate.py  — Word 파일 생성기 (공유)
```

---

## 트리거 패턴 (bot.py)

```python
_WORD_ORG_RE = re.compile(r"워드\s*정리", re.IGNORECASE)
```

---

## 상태 관리

- `_word_org_state[chat_id] = True` : 파일 대기 중
- 파일 수신 후 자동 초기화
- "워드 정리" 재입력 시 → 상태 재설정 (재시작)
